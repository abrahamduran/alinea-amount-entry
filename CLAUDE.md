## Current state vs. plan

The project was just generated by Xcode (`AlineaApp.swift` + stock `ContentView.swift` + empty test files). **No feature code, design system, fonts, or custom assets exist yet.** When implementing, follow the structure in §5 of the plan.

Target is **iOS 26.2 / Swift 6** (strict concurrency). iOS 26 was chosen deliberately to unlock Liquid Glass (`.glassEffect()`, `GlassEffectContainer`, `.buttonStyle(.glass)`) for the AUTOMATED badge backdrop and back-chevron container, and the `@Animatable` macro for the Allie halo.

## Architecture (target shape, from the plan)

- **One SPM package `FoundationKit`** under `Packages/`, exposing a `DesignSystem` library target (colors, typography, fonts, brand gradient, shared primitives like `BackButton` / `AutomatedBadge`). Named "Kit" so future siblings (`Networking`, `Persistence`) fit without restructuring.
- **Feature code lives directly in the app target** under `App/Features/AmountEntry/`. Over-packaging a single screen is ceremony.
- **`AmountInputModel`** is a pure value type holding all input rules — `append(digit:)`, `appendDecimal()`, `backspace()`, `set(amount:)`, plus derived `isEmpty` / `canReview` / `isDecimalDisabled`. Tested directly without SwiftUI.
- **`AmountEntryViewModel`** is `@Observable @MainActor`, wraps the input model, exposes display-ready surface state, and bumps a `tapCount` to drive `.sensoryFeedback`.
- **`AmountFormatter`** is custom (not `Decimal.FormatStyle.currency`) because it caps at 2 decimals **and** preserves the in-progress shape — a freshly-typed trailing dot (`$2,000.`) or a single decimal digit (`$2,000.5`) that `FormatStyle.currency` would normalize away by padding to `.00`. Locale-aware integer grouping comes for free via `Int.formatted(.number)`.

## Design system conventions

- Asset catalog uses **namespaced folders** with "Provides Namespace" on (`Background/`, `Text/`, `Brand/`) — dark-only, no light-mode mappings.
- Every color is accessed via semantic enums on `Color` (e.g. `Color.Background.screen`, `Color.Text.placeholder`, `LinearGradient.brandGradient`). **No inline hex anywhere in feature code.**
- Typography is a `Typography` enum + `.typography(_:)` view modifier — single source of truth for font, tracking, text-case, and Dynamic Type anchor. Don't chain `.font()` + `.tracking()` at call sites.
- Custom fonts (`GT Flexa Condensed Medium`, `Instrument Sans SemiCondensed Medium`) are registered at app launch via `CTFontManagerRegisterFontsForURL`, AND listed in the app target's `Info.plist` `UIAppFonts` so Previews render correctly. Use the font's **PostScript name** (verify in Font Book), not the file name.

## Behavioral rules (the 7 Figma comments — non-negotiable)

1. Chips visible only when amount is empty; Review button only when filled.
2. Review button gradient must animate (open choice — see plan §7 "Allie glow").
3. **Chips → Review transition is the headline differentiator** — Ryan explicitly called it out. Spend disproportionate polish time here.
4. Haptics on every keypad tap (`.sensoryFeedback`, not `UIImpactFeedbackGenerator`).
5. Caret blinks and always sits at the end, in both empty and filled states.
6. Decimal key disables (visually + functionally) when input already contains `.`.
7. Amount text scales to fit (`.minimumScaleFactor(0.3)`).

Exact input-rule pseudocode, max-decimal cap (4, per Ryan), and leading-zero/backspace edge cases are in plan §4.

## Hard constraints

- **SwiftUI only** — zero `import UIKit` anywhere.
- **No third-party deps** other than `pointfreeco/swift-snapshot-testing` for snapshot tests.
- **No String Catalog** — localization isn't being evaluated; hard-code English.
- Light mode is not designed — don't fabricate it.
- Accessibility is "nice-to-have" only — ~30 min of basic labels, don't go deeper (per Ryan).
